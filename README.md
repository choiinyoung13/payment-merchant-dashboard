## 🛠 기술 스택

- **프레임워크**: Next.js 16.0.3 (App Router)
- **언어**: TypeScript 5.x
- **UI 라이브러리**: React 19.2.0
- **스타일링**: Tailwind CSS 4.x
- **차트 라이브러리**: Recharts 3.4.1
- **패키지 매니저**: npm

## 📦 개발 환경 요구사항

- **Node.js**: 20.x LTS (필수)

  - 예: `20.11.0`, `20.15.0` 등
  - Node 18 이하, 22 이상 버전 사용 시 발생하는 문제는 지원자 측에서 해결해야 합니다.

- **패키지 매니저**: npm (Node 기본 포함 버전)

## 🚀 설치 및 실행

### 1. 의존성 설치

```bash
npm install
```

### 2. 개발 서버 실행

```bash
npm run dev
```

개발 서버가 실행되면 브라우저에서 [http://localhost:3000](http://localhost:3000)을 열어 확인할 수 있습니다.

### 3. 프로덕션 빌드 및 실행

```bash
npm run build
npm start
```

## ✨ 주요 기능

### 1. 대시보드 페이지 (`/`)

- **주요 지표 카드**

  - 총 거래 건수 (상태별 분류: 완료/실패/취소)
  - 총 거래액 (기간 선택 가능)
  - 활성 가맹점 수
  - 최다 브랜드
  - 최다 업종

- **통계 시각화**

  - 결제 수단별 통계 (Progress Bar)
  - 가맹점 상태/업종별 분포 (Bar Chart)

- **최근 거래 내역**

  - 최근 5건의 거래 내역 표시
  - 상태별 색상 구분

- **Top 가맹점**
  - 금액 기준 Top 5 가맹점
  - 건수 기준 Top 5 가맹점

### 2. 거래 내역 페이지 (`/payments`)

- **필터링 및 정렬**

  - 기간 선택 (월 단위)
  - 결제 상태 필터 (완료/실패/취소)
  - 결제 수단 필터
  - 금액 정렬
  - 검색 (결제코드 또는 가맹점명)

- **요약 카드**

  - 총 거래 건수
  - 총 거래액
  - 최다 거래액 가맹점

- **통계 시각화**

  - 결제 수단별 통계 (Progress Bar)

- **거래 내역 리스트**
  - 반응형 디자인 (모바일: 카드 형태, 데스크톱: 리스트 형태)
  - 상태별 아이콘 배지
  - 24시간 형식 시간 표시

### 3. 가맹점 목록 페이지 (`/merchants`)

- **필터링 및 검색**

  - 상태 필터 (활성/비활성/준비/종료)
  - 검색 (가맹점명 또는 가맹점 코드)

- **요약 카드**

  - 총 가맹점 수
  - 활성 가맹점 수
  - 최다 업종
  - 최다 브랜드

- **통계 시각화**

  - 가맹점 상태별 분포 (Bar Chart)
  - 업종별 분포 (Bar Chart)

- **가맹점 리스트**
  - 상태별 배지 표시
  - 가맹점 상세 정보 모달 (바텀 시트)

## 🎨 디자인 의도 및 주요 UI/UX 포인트

### 디자인 철학

1. **일관성 있는 디자인 시스템**

   - 메인 컬러 `#425aeb`를 통한 브랜드 일관성 유지
   - 모든 페이지에서 동일한 컴포넌트 스타일 적용 (SummaryCard, BottomSheet 등)
   - 통일된 간격, 폰트 크기, 색상 팔레트 사용

2. **모바일 퍼스트 반응형 디자인**

   - 최소 너비 320px부터 지원
   - 390px, 540px, 768px, 1024px 등 주요 브레이크포인트에서 최적화
   - 모바일에서는 카드 형태, 데스크톱에서는 리스트 형태로 자동 전환

3. **직관적인 사용자 경험**

   - 바텀 시트 모달을 통한 일관된 필터링 UI
   - 상태별 색상 구분으로 빠른 정보 파악
   - 검색과 필터를 한 곳에서 관리하여 접근성 향상

4. **정보 밀도 최적화**
   - 요약 카드와 상세 리스트의 명확한 구분
   - 통계 시각화를 통한 데이터 인사이트 제공
   - 불필요한 정보 제거로 핵심 정보에 집중

### 주요 UI/UX 특징

- **바텀 시트 모달**: 모바일 친화적인 필터링 인터페이스
- **상태 배지**: 색상과 텍스트로 상태를 즉시 파악 가능
- **반응형 그리드**: 화면 크기에 따라 자동으로 레이아웃 조정
- **호버 효과**: 인터랙티브한 요소에 시각적 피드백 제공
- **24시간 형식**: 시간 표시의 명확성 향상

## 🔌 API 설정

### API Base URL

프로젝트는 다음 API 서버를 사용합니다:

- **Base URL**: `https://recruit.paysbypays.com/api/v1`
- **Swagger 문서**: [https://recruit.paysbypays.com/swagger-ui/index.html](https://recruit.paysbypays.com/swagger-ui/index.html#/)

### API 설정 위치

API Base URL은 `lib/api.ts` 파일에 하드코딩되어 있습니다:

```typescript
export const API_BASE_URL = 'https://recruit.paysbypays.com/api/v1'
```

### Mock 데이터

개발 환경에서는 일부 API가 로컬 Mock 데이터를 사용합니다:

- 거래 내역: `http://localhost:3000/api/v1/payments/list` (Next.js API Route)
- 가맹점 목록: 실제 API 서버 사용
